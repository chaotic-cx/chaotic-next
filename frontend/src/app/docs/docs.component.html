You can always report build failures or broken packages via our GitLab or GitHub repositories by creating a new issue.

Our main repository is the one on GitLab as the build system makes extensive use of its CI pipelines.

If you would like to see a new package added to Chaotic-AUR, please submit a new issue via our GitHub repository. Issues on GitLab are more likely to be ignored. We prefer for historical reasons.


Before submitting the request, ensure searching for existing issues concerning the same package.
We might have been dealing with it already in the past or removed it for a reason.


Also, check whether the license permits distributing the software, as well as it not being part of our ban list.
